What to do with the new leaf taggings? (What it's about?)

Example: assume you have the following tags tree: 

  ├── foo
  │   └── subfoo
  └── bar

And you have a bookmark X which is tagged with /foo/subfoo. Now, you're going to move subfoo under bar, so that the new tags tree will be: 

  ├── foo
  └── bar
      └── subfoo

The question is: how your bookmark X should be tagged after that? There are two options:

  • Delete new tagging leaf: the bookmark X will be tagged with /bar/subfoo
  • Keep new tagging leaf: the bookmark X will be tagged with /foo and /bar/subfoo

Note: the tag /foo will stay in any case, we're talking about taggings: the linkage between tags and bookmarks.



What to do with the new leaf taggings? (What it's about?)

Example: assume you have the following tags tree: 

  ├── foo
  │   └── subfoo
  └── bar

And you have a bookmark X which is tagged with /foo/subfoo and /bar. Now, you're going to delete subfoo, so that the new tags tree will be: 

  ├── foo
  └── bar

The question is: how your bookmark X should be tagged after that? There are two options:

  • Delete new tagging leaf: the bookmark X will be tagged with /bar.
  • Keep new tagging leaf: the bookmark X will be tagged with /foo and /bar.

Note: the tag /foo will stay in any case, we're talking about taggings: the linkage between tags and bookmarks.
